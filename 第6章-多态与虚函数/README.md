# 第6章 多态与虚函数

> 本章目标：理解为什么“同一个函数调用”会在运行时表现出不同效果。多态就是同一句话，不同对象有不同回答。

## 1. 多态是什么

| 类型 | 时机 | 例子 |
| --- | --- | --- |
| 编译时多态 | 编译时确定 | 函数重载、运算符重载、模板 |
| 运行时多态 | 运行时确定 | 虚函数 |

运行时多态常见写法：基类指针或引用指向派生类对象。

## 2. 虚函数

在基类函数前加 `virtual`，表示允许派生类重写，并支持运行时绑定。

```cpp
class Animal {
public:
    virtual void speak() const {
        cout << "普通声音";
    }
};
```

## 3. override 与 final

`override` 告诉编译器：“我就是要重写基类虚函数。”如果写错，编译器会提醒。

```cpp
class Cat : public Animal {
public:
    void speak() const override {
        cout << "喵";
    }
};
```

`override` 是好习惯，像交作业前让编译器帮你查格式。

## 4. 纯虚函数和抽象类

```cpp
class Shape {
public:
    virtual double area() const = 0;
};
```

包含纯虚函数的类叫抽象类，不能直接创建对象，只能作为接口或基类。抽象类像菜单，不是菜本身；真正能端上桌的是派生类。

## 5. 虚析构函数

只要一个类准备用作多态基类，析构函数就应该是虚的。

```cpp
class Base {
public:
    virtual ~Base() = default;
};
```

否则用基类指针删除派生类对象时，派生类析构可能不会被正确调用。

## 6. 多态的实现直觉

有虚函数的对象通常会带着一张“函数地址表”的线索。调用虚函数时，程序会根据对象真实类型找到对应实现。

你不必一开始就背虚函数表细节，但要记住：

- 必须通过基类指针或引用调用，才体现运行时多态。
- 基类函数要写 `virtual`。
- 派生类重写建议写 `override`。

## 7. 本章小结

- 多态让统一接口拥有多种实现。
- `virtual` 开启运行时绑定。
- `override` 能减少低级错误。
- 抽象类定义接口，派生类负责具体实现。
- 多态基类请写虚析构函数。
