## 1. 数据结构概述

数据结构关心三件事：数据之间是什么关系、数据在内存里怎么存、增删改查的代价是多少。

| 复杂度 | 感觉 |
| --- | --- |
| `O(1)` | 一步到位 |
| `O(log n)` | 每次砍一半，比如二分查找 |
| `O(n)` | 一个个看 |
| `O(n^2)` | 双层循环，数据大了会很累 |

## 2. 顺序表

顺序表通常用数组实现，内存连续。优点是按下标访问快，缺点是中间插入和删除要搬家。

```cpp
vector<int> nums = {1, 2, 3};
nums.push_back(4);
cout << nums[0];
```

顺序表像电影院座位，知道第几排第几座就能直接找到；但要在中间加一个人，后面一排都得挪。

## 3. 链表

链表由结点组成，每个结点保存数据和下一个结点的地址。

```cpp
struct Node {
    int data;
    Node* next;
};
```

链表像一串接力棒，每个人只知道下一个人在哪。找第 100 个，就老老实实传 100 次。

## 4. 栈与队列

栈：后进先出。队列：先进先出。

```cpp
stack<int> s;
s.push(1);
s.pop();

queue<int> q;
q.push(1);
q.pop();
```

栈像摞盘子，最后放上去的先拿；队列像排队买奶茶，先来先服务。

## 5. 查找算法

顺序查找从头到尾找，简单但慢。二分查找要求数据有序，每次砍掉一半。

```cpp
int l = 0, r = nums.size() - 1;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (nums[mid] == target) break;
    if (nums[mid] < target) l = mid + 1;
    else r = mid - 1;
}
```

## 6. 排序算法

| 算法 | 思路 | 平均复杂度 |
| --- | --- | --- |
| 冒泡排序 | 大的慢慢往后冒 | `O(n^2)` |
| 选择排序 | 每轮选最小的 | `O(n^2)` |
| 快速排序 | 分治，左右分区 | 平均 `O(n log n)` |

实际写程序时，多数情况下直接用：

```cpp
sort(nums.begin(), nums.end());
```

## 7. 动态内存与复杂度

`new` 出来的东西要 `delete`，`new[]` 对应 `delete[]`。

```cpp
int* a = new int[10];
delete[] a;
```

现代 C++ 更推荐用 `vector`、`string`、智能指针减少手动管理。

## 8. 本章小结

- 顺序表访问快，插入删除慢。
- 链表插入删除灵活，随机访问慢。
- 栈后进先出，队列先进先出。
- 二分查找必须基于有序数据。
- 能用 STL 容器时，不要急着手搓轮子。
