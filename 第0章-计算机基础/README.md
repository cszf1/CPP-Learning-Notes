## 1. 计算机系统组成

计算机系统由硬件和软件组成。硬件像身体，软件像指挥身体做事的计划书。

常见硬件：

- CPU：负责运算和控制，是“总调度”。
- 内存：程序运行时临时放数据，断电就忘事。
- 外存：硬盘、U盘等，适合长期保存。
- 输入输出设备：键盘、显示器、网卡等。

软件分为系统软件和应用软件。操作系统负责管理硬件资源，程序员写的程序通常运行在操作系统之上。

## 2. 冯·诺依曼结构

核心思想：程序和数据都存放在存储器里，CPU 按顺序取指令、解释指令、执行指令。

五大部件：

| 部件 | 作用 |
| --- | --- |
| 运算器 | 做算术和逻辑运算 |
| 控制器 | 指挥各部件协同工作 |
| 存储器 | 保存程序和数据 |
| 输入设备 | 输入信息 |
| 输出设备 | 输出结果 |

程序运行像做菜：菜谱是程序，食材是数据，厨师是 CPU，厨房柜子是存储器。厨师一边看菜谱一边处理食材。

## 3. 存储器与总线

存储器有层次：寄存器最快但最小，缓存很快，内存较大，硬盘最大但慢。

| 层次 | 特点 |
| --- | --- |
| 寄存器 | CPU 内部，速度最快 |
| Cache | 缓存热点数据 |
| 内存 RAM | 程序运行主战场 |
| 外存 | 长期保存数据 |

总线负责传输信息，常见有地址总线、数据总线、控制总线。地址总线像门牌号，数据总线像搬货通道，控制总线像“现在开始搬”的口令。

## 4. 数制转换

C++ 程序里常见进制：

| 进制 | 示例 | 说明 |
| --- | --- | --- |
| 二进制 | `0b1010` | 逢 2 进 1 |
| 八进制 | `012` | 逢 8 进 1 |
| 十进制 | `10` | 日常最常用 |
| 十六进制 | `0xA` | 逢 16 进 1 |

二进制和十六进制转换很方便：4 位二进制正好对应 1 位十六进制。

## 5. 机器数与补码

计算机中整数通常用补码表示。原因很实用：加法器一套电路就能处理加法和减法，机器少费劲，人类少掉头发。

简单记法：

- 正数：原码、反码、补码相同。
- 负数补码：绝对值写成二进制，按位取反，再加 1。
- n 位补码范围：`-2^(n-1)` 到 `2^(n-1)-1`。

浮点数用于表示小数，但不是所有十进制小数都能精确表示。所以 `0.1 + 0.2` 可能不是精确的 `0.3`。这不是电脑叛逆，是二进制表示能力有限。

## 6. 字符编码

ASCII 用 1 个字节表示常见英文字符。中文字符更多，需要更大的编码空间。Unicode 是字符集合，UTF-8 是常见编码方式。

重点记住：字符在计算机里也是数字。`'A'` 的 ASCII 值是 65，字符比较本质上常常是在比较编码值。

## 7. 本章小结

- 程序最终会变成机器指令执行。
- CPU、内存、外存各有分工。
- 补码让减法也能用加法思路处理。
- 浮点数有精度问题，比较小数时别太天真。
- 字符编码决定“字符如何变成数字”。
